GET 방식으로 요청 후 응답페이지 받아보기 테스트

특징1. GET방식으로 요청하는건 URL의 Header 영역에 데이터들을 포함시켜서 요청함
=>사용자가 입력한 값(데이터)들이 URL에 노출됨
=>보안유지가 불가능함
=>즉, 로그인이나 회원가입 등과 같은 경우 GET 방식이 부적합함

특징2. Header 영역은 전송하는 길이에 제한이 있음
=>방대한 데이터를 담았을 경우 초과된 데이터는 그대로 절단되서 넘어감
=>즉, 게시판 작성 같은 경우도 GET 방식이 부적합함

특징3. 장점이라고 한다면 URL에 데이터가 노출되기 때문에 즐겨찾기 기능이 가능하다는 점이 있음 (북마크==즐겨찾기)
=>즉, 검색 기능 같은 경우 GET 방식이 더 적합함
